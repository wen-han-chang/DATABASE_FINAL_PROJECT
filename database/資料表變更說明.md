# 資料表變更說明

> 比對「目前實際在用的資料庫」與最初的 `database/schema.sql`，
> 整理出目前 schema 與資料同步流程的變更，方便工程團隊銜接。

## 一句話結論

**結構上只多了一張表：`stock_sync`。** 原本的 13 張表（users / investor_profiles /
sectors / stocks / stock_daily_bars / portfolios / holdings / orders / decision_cards /
decision_card_tags / knowledge_base / knowledge_sentences / knowledge_related_tags）
**欄位、型別、關聯都沒有變**。

## 1. 新增的表：`stock_sync`

| 欄位 | 型別 | 說明 |
|------|------|------|
| `stock_id` | SMALLINT，PK，FK→stocks(id) | 哪一檔股票 |
| `source` | NVARCHAR(20)，預設 `'seed'` | `seed`=初始測試資料；`twse`=完整真實歷史就緒 |
| `last_synced` | DATETIME2(0)，預設現在時間 | 上次同步時間 |

**為什麼要這張表**：K 線歷史改成「資料庫當快取、TWSE 當真相來源」。
這張表記錄每檔「上次有沒有、什麼時候從 TWSE 抓過真實歷史」，後端據此判斷
要不要重抓，避免每次都打官方 API。

**要不要自己去 SSMS 建？不用。**
後端 `backend/db.js` 啟動時會自動偵測並建立這張表（`ensureSchema`）。
為了交接完整，`schema.sql` 也已補上這段（`IF OBJECT_ID ... IS NULL CREATE TABLE`），
手動重建整份 schema 時也會一起建。**兩種方式都安全、不會重複建。**

## 2. 資料內容的行為變化（不是結構變化，但要知道）

- `seed.js` 一開始灌的 `stock_daily_bars` 是**測試**資料。
- 只要有人在前端看某一檔，後端會去 TWSE 官方抓那一檔的**真實歷史日線**，
  把該檔的測試資料**整批換成真實資料**，並把 `stock_sync.source` 標成 `twse`。
- 所以隨著使用，`stock_daily_bars` 會逐檔從「測試資料」變「真實資料」。
- 想整個清掉重來：點專案根目錄 `clear-stock-data.bat`（需輸入 `yes`），
  它會清空 `stock_daily_bars` 與 `stock_sync`（不動帳號/交易/股票目錄），
  之後再被瀏覽時會重新抓真實資料。
- 若使用 `database/schema.sql` 的整庫重置區塊，已包含 `stock_sync`；
  重置順序必須先刪 `stock_sync`，再刪 `stocks`，避免外鍵參考造成 DROP 失敗。

## 3. 容易誤會的點：交易單位「股 vs 張」

- 前端畫面顯示與輸入是「股 (shares)」。
- 但**資料庫 `holdings` / `orders` 仍是用「張 (lots)」欄位，結構沒有改**。
- 換算是在前端做的（1 張 = 1000 股）。
- 所以查資料庫看到 `lots` 是正常的，不是漏改。

## 4. 共用資料庫提醒

`ncu_db` 是共用的，裡面還有**別專案的表**（例如 `calendar`、`calendar02`、
`stock_data`、`stock_list`、`Candlestick_chart_def`、`tradingPrice_def`、
`tradingVolume_def`、`year_calendar`）。**那些不是本專案的，不要動它們。**
本專案只用：上面列的 13 張 + `stock_sync`，共 14 張。

## 5. 怎麼確認自己的資料庫對得上

任一種即可：
1. 直接啟動後端（會自動補建 `stock_sync`）。
2. 或在 SSMS 重跑 `database/schema.sql`（冪等，可重複執行，不會刪資料）；
   它最後的 Section 5 驗證會列出本專案 **14 張**表。

## 6. 變更檔案一覽

| 檔案 | 變更 |
|------|------|
| `database/schema.sql` | 新增 `stock_sync` 建表段（Section 2 表 14）＋ Section 1 重置區塊先刪 stock_sync ＋ Section 5 驗證清單加入 stock_sync |
| `backend/db.js` | 啟動時 `ensureSchema` 自動補建 `stock_sync` |
