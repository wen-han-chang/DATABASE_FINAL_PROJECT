# 資料庫連線設定說明

本文件說明「投智 AI」後端如何連到 SQL Server 資料庫（`ncu_db`），
以及**換人/換機器/接別人的資料庫時要改哪裡**。

---

## 1. 連線資訊放在哪裡

所有連線設定都集中在 **`backend/.env`** 這一個檔案，程式碼裡不寫死帳密。

- `backend/.env`：真實設定檔，**已被 `.gitignore` 排除，不會上傳 git**。
- `backend/.env.example`：範例檔，給別人看格式用，**不放真實密碼**。
- 讀取機制：`backend/loadEnv.js` 在 server 啟動時把 `.env` 讀進 `process.env`
  （因為目前 Node 是 v18.1.0，不支援 `node --env-file`，故自寫極簡載入器）。

---

## 2. ⚠️ 若要「接別人的資料庫」請替換這一段 ⚠️

> **這是本文件最重要的部分。**
> 換組員的電腦、換成助教/雲端、或接別人架好的 SQL Server 時，
> **只需要改 `backend/.env` 裡的 `DB_*` 區塊，其他程式完全不用動。**

打開 `backend/.env`，把下面這段整批換成「對方提供的連線資訊」：

```env
# ── 若接別人的資料庫，替換以下整段 ───────────────
DB_HOST=127.0.0.1          # ← 改成對方的主機 IP 或網域
DB_PORT=1433               # ← 改成對方的 SQL Server 埠號（預設 1433）
DB_USER=skyfire            # ← 改成對方給的帳號
DB_PASSWORD=1487           # ← 改成對方給的密碼
DB_NAME=ncu_db             # ← 改成對方的資料庫名稱
DB_ENCRYPT=false           # ← 本機 false；連雲端(Azure)改 true
DB_TRUST_CERT=true         # ← 本機自簽憑證 true；正式憑證可改 false
# ───────────────────────────────────────────────
```

改完存檔，重啟後端（`Ctrl+C` 後重新 `npm run dev`）即生效。
**不需要改任何 `.js` 程式碼。**

> 目前專案內建的這組（`skyfire` / `1487` / `127.0.0.1` / `ncu_db`）
> 是本機開發用。交付或換人時務必確認這段是否要替換。

## 3. 全新環境的安裝步驟

第一次在一台新電腦跑後端：

1. 安裝依賴
   ```bash
   cd backend
   npm install
   ```
2. 建立設定檔（複製範例再填值）
   ```bash
   # Windows PowerShell
   Copy-Item .env.example .env
   ```
   然後打開 `backend/.env`，依第 2 節填入正確的 `DB_*`。
3. 在 SSMS 建立資料表
   - 用你的帳號登入 SSMS。
   - 開啟 `database/schema.sql`。
   - 按 **執行 (F5)**。
   - 看「結果」視窗應出現 **13 列、每列 row_count = 0**，代表 13 張表建好。
4. 啟動後端
   ```bash
   npm run dev
   ```
5. 驗證連線（見第 4 節）。

## 4. 如何驗證連線成功

後端啟動後，瀏覽器或 curl 開啟：

```
http://localhost:3001/api/health
```

正常會回傳：

```json
{
  "ok": true,
  "db": { "connected": true, "name": "ncu_db", "error": null }
}
```

- `db.connected: true` → 後端**確實連到資料庫**。
- `db.connected: false` → 連線失敗，`db.error` 會告訴你原因，對照第 5 節。

## 5. 常見錯誤與排除

| 錯誤訊息關鍵字 | 原因 | 解法 |
|---|---|---|
| `ELOGIN` / `Login failed for user` | 帳號或密碼錯，或 SQL Server 沒開 SQL 驗證 | 檢查 `.env` 的 `DB_USER`/`DB_PASSWORD`；在 SSMS 把伺服器設成「SQL Server 及 Windows 驗證模式」後重啟 SQL Server 服務 |
| `ESOCKET` / `ETIMEOUT` / 連不到 | SQL Server 沒開、TCP/IP 沒啟用、防火牆擋 1433 | 開「SQL Server 組態管理員」啟用 TCP/IP；確認 SQL Server 服務在跑；放行 1433 |
| `Cannot open database "ncu_db"` | 資料庫還沒建或名稱錯 | 在 SSMS 確認 `ncu_db` 存在；或先跑 `database/schema.sql`（Section 0 會自動建庫） |
| `self signed certificate` | 加密設定不符 | `.env` 設 `DB_ENCRYPT=false`、`DB_TRUST_CERT=true` |
| 中文存進去變 `????` | 欄位不是 NVARCHAR 或字面值漏 `N''` | 用 `database/schema.sql` 建表（已全部用 NVARCHAR）；自寫 SQL 插中文要加 `N'中文'` |

## 6. 安全與注意事項

- **絕對不要把 `backend/.env` 上傳 git**。它已在 `.gitignore`，但 commit 前仍請確認。
- 真實密碼只放 `.env`；`.env.example` 與所有 `.js`、`.md` 只放佔位字或本機開發值。
- 環境相容性：目前 Node `v18.1.0`，故 `mssql` 鎖在 `v10`（官方支援 Node 14+）。
  若日後升級 Node 20+，可改回 `mssql@^11`。**升級 Node 前請先和組員確認**，
  以免別人的環境跑不動。
- `npm install` 時出現的 3 個 moderate 漏洞屬交付鏈相依，本機教學專題可接受；
  **不要**執行 `npm audit fix --force`（會做破壞性升級，可能弄壞依賴）。
