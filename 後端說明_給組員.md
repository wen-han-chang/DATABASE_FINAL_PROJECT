## 2. 系統架構與資料流

```
前端 Vue (5173)
   │  fetch + Bearer token
   ▼
後端 Node http server (3001)  ── server.js（路由）
   │                          ── auth.js（密碼雜湊 / token）
   │                          ── dao.js（業務 SQL + 交易）
   ▼
SQL Server  ncu_db (1433)     ── 13 張資料表
```

- 後端**零框架**（沿用原本 Node 內建 `http`，沒上 Express），維持「看得懂、好維護」。
- 前端只透過後端 API 拿資料，不直接碰資料庫。

## 5. 為什麼買賣要包「資料庫交易」（demo 重點）

一筆買單同時要做三件事：

1. `orders` 寫一筆委託紀錄
2. `holdings` 新增或更新持股（重算平均成本）
3. `portfolios.cash` 扣款

只要中間任一步失敗（例如扣了錢卻沒記持股），資料就壞了。
`dao.js` 用 `withTransaction` 包起來：**全部成功才 commit，任一步失敗整批 rollback**。
這是資料庫課程最該展示的點，demo 時可現場開 SSMS 查 `orders` 看資料即時進出。

## 6. 前端改了什麼（三個 store）

前端畫面與元件**沒大改**，只把三個 Pinia store 從「假資料」改成「打後端 API」，
方法簽名全部保留，所以其他元件不受影響：

- `stores/auth.js`：登入改打 `/api/auth/login`，存 token
- `stores/investor.js`：問卷改存 / 讀後端
- `stores/portfolio.js`：設資產 / 買 / 賣 / 重置改打後端
- 新增 `services/api.js`：統一封裝上述呼叫與 token

## 8. 組員務必注意事項

1. **換連線 / 接別人的資料庫**：只改 `backend/.env` 的 `DB_*` 區塊，**程式不用動**。
   詳細看 `database/資料庫連線設定.md` 第 2 節（有逐欄說明）。
2. **`backend/.env` 不會上傳 git**（已在 `.gitignore`）。每個人要自己建一份。
   git 上只有 `backend/.env.example` 範例。**不要把真實密碼貼進任何 .js / .md / .example**。
3. **Node 版本**：目前用 Node 18，`mssql` 已鎖在 v10。要升級 Node 前先全組講好，
   不然別人環境會跑不動。
4. **K 線是模擬資料**：正式報告呈現真實行情前要替換，不可當真。
5. **資料庫測試殘留**：開發時測過幾個測試帳號（`test1@ncu.edu` 等）還在 `users` 表，
   不影響 demo；要清需手動 `DELETE`。`node seed.js --force` 只重灌行情三張表、
   **不會**動帳號（且若已有交易會被外鍵擋住，這是正確保護）。
6. **重複跑 seed 安全**：已灌過會自動略過，不會重複塞。
