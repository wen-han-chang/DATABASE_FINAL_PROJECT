# 給前端組員的後端串接說明

> 閱讀對象：負責 Vue 前端畫面、元件、store、service 的組員。
> 這份文件只說明「backend 對前端提供了什麼」以及「前端要怎麼呼叫」。
> 專案怎麼啟動、要開幾個終端機、demo 怎麼跑，請看同資料夾的《給全組的專案啟動操作手冊.md》。

---

## 先看這段：後端不是只用一個 TWSE 網站

本專案有使用 `https://openapi.twse.com.tw/`，但它只負責適合它的資料，不負責全部股票資料。

| 前端想要什麼 | 後端用哪裡抓 | 前端要怎麼用 |
|--------------|--------------|--------------|
| 股票清單、最新交易日資料、市場驗證 | `https://openapi.twse.com.tw/` | 可用於清單、驗證面板、最新交易日資料 |
| 歷史 K 線與 MA 指標 | `https://www.twse.com.tw/exchangeReport/STOCK_DAY` | 呼叫 `getDbBars(code)`，用回傳的 OHLCV 畫 K 線 |
| 準即時現價、漲跌、開高低量 | `https://mis.twse.com.tw/stock/api/getStockInfo.jsp` | 呼叫 `getQuote(code)`，顯示在 K 線圖上方 |

前端要記住一件事：**K 線圖用歷史日線，報價條用準即時報價，兩個可以放在同一個畫面，但不要混成同一筆資料。**

後端這樣設計是為了避免首頁一次抓全部股票即時報價，也避免把盤中價格誤當成正式日 K 收盤資料。

---

## 1. 目前前端應該怎麼理解後端

前端不應該直接呼叫 TWSE 官方 API。正確流程是：

```txt
前端 Vue 元件
  -> frontend/src/services/twseApi.js
  -> http://localhost:3001
  -> backend
  -> SQL Server 或 TWSE 官方資料來源
```

這樣設計的原因：

- 前端不用知道 TWSE 官方 API 欄位長什麼樣。
- 前端不用處理 CORS、資料格式轉換、錯誤重試。
- 後端可以控制資料庫寫入、快取、同步頻率。
- 之後如果資料來源更換，前端只需要維持呼叫同一組 service。

目前前端最重要的兩個股票資料來源：

| 前端需求 | 要呼叫的 service | 後端 route | 資料來源 |
|----------|------------------|------------|----------|
| K 線圖與 MA 指標 | `getDbBars(code)` | `/api/market/db-bars/:code` | SQL Server；必要時由後端向 TWSE `STOCK_DAY` 抓歷史日線 |
| 圖表上方準即時報價 | `getQuote(code)` | `/api/quote/:code` | TWSE MIS 單檔報價，後端 20 秒快取 |

## 2. 重要觀念：歷史 K 線與即時報價要分開

前端畫面可以把兩者放在同一個股票圖表區，但資料流不能混在一起。

建議畫面結構：

```txt
股票圖表區
  -> 上方：準即時報價條，顯示現價、漲跌、開高低量、時間
  -> 下方：歷史 K 線圖，使用日線資料計算 MA 與週 K/月 K
```

為什麼要分開：

- 歷史 K 線是正式交易日資料，例如一整天的開盤、最高、最低、收盤、成交量。
- 準即時報價只是使用者當下看到的最新狀態，還不是正式日 K。
- 如果把準即時價格硬塞進最後一根 K 棒，MA 指標會被盤中價格影響，容易誤導。
- 分開之後比較好向老師說明：K 線負責歷史分析，報價條負責當下價格。

## 3. 前端 service 檔案

前端統一從這個檔案呼叫後端：

```txt
frontend/src/services/twseApi.js
```

主要函式：

| 函式 | 用途 | 目前建議 |
|------|------|----------|
| `getDbBars(code)` | 取得某檔股票歷史 K 線資料 | K 線圖優先使用 |
| `getQuote(code)` | 取得某檔股票準即時報價 | 報價條使用 |
| `getMappedStocks()` | 取得 TWSE OpenAPI 整理後的股票清單 | 可用於列表或驗證 |
| `getMappedStockByCode(code)` | 取得單檔最新交易日資料 | 可用於基本資訊 |
| `getMappedDailyBars(query)` | 取得最新交易日 OHLCV | 不是長期 K 線主來源 |
| `getMappedDailyBarByCode(code)` | 取得單檔最新交易日 OHLCV | 不是長期 K 線主來源 |
| `getMarketImportPreview()` | 預覽可匯入資料庫的格式 | 給後端或資料庫檢查 |
| `getMarketIndex()` | 取得 TWSE 指數資料 | 給驗證面板使用 |
| `getTwseRaw(path, query)` | 透過後端 proxy 查 TWSE 原始資料 | 除錯或補充用 |

重要規則：

- 前端元件不要散落寫 `fetch('http://localhost:3001/...')`。
- 如果要新增 route，先在 `twseApi.js` 包成函式，再讓元件呼叫。
- 這樣後端 route 改名或資料格式微調時，前端只需要集中改一個地方。

## 4. K 線圖目前使用方式

目前主要元件：

```txt
frontend/src/components/StockChart.vue
```

資料流程：

```txt
StockChart.vue
  -> getDbBars(stock.code)
  -> GET /api/market/db-bars/:code
  -> 後端回傳歷史 OHLCV
  -> StockChart.vue 計算日 K、週 K、月 K、MA 指標
  -> 畫在同一張 K 線圖
```

呼叫範例：

```js
import { getDbBars } from '@/services/twseApi'

const response = await getDbBars('2330')

if (response.ok && Array.isArray(response.data)) {
  const bars = response.data
}
```

後端回傳格式重點：

```json
{
  "ok": true,
  "source": "ncu_db.stock_daily_bars",
  "code": "2330",
  "count": 253,
  "data": [
    {
      "date": "2025-05-02",
      "open": 968,
      "high": 968,
      "low": 959,
      "close": 963,
      "volume": 28923
    }
  ]
}
```

前端應使用的欄位：

- `date`：K 線日期。
- `open`：開盤價。
- `high`：最高價。
- `low`：最低價。
- `close`：收盤價。
- `volume`：成交量。

如果 `data` 是空陣列：

- 代表後端找不到這檔股票，或 TWSE 當下抓不到資料。
- 目前前端可以退回模擬資料當備援，但畫面要標示清楚，不要假裝是真實歷史。

## 5. 準即時報價目前使用方式

目前主要元件：

```txt
frontend/src/components/LiveQuote.vue
```

資料流程：

```txt
LiveQuote.vue
  -> getQuote(stock.code)
  -> GET /api/quote/:code
  -> 後端檢查 20 秒快取
  -> 沒快取才向 TWSE MIS 抓單檔報價
  -> 回傳現價、漲跌、開高低量、時間
  -> 顯示在 K 線圖上方
```

呼叫範例：

```js
import { getQuote } from '@/services/twseApi'

const response = await getQuote('2330')

if (response.ok) {
  const quote = response.data
}
```

後端回傳格式重點：

```json
{
  "ok": true,
  "data": {
    "code": "2330",
    "name": "台積電",
    "price": 2265,
    "prevClose": 2270,
    "open": 2310,
    "high": 2325,
    "low": 2250,
    "volume": 29774,
    "time": "13:30:00",
    "date": "20260515",
    "change": -5,
    "changePct": -0.22,
    "closed": false,
    "cached": false
  }
}
```

前端顯示建議：

- `price`：顯示現價。
- `change` 與 `changePct`：顯示漲跌與漲跌幅。
- `open`、`high`、`low`、`volume`：顯示今日開高低量。
- `time`：顯示報價時間，避免使用者以為資料一定每秒更新。
- `cached`：可以不用顯示給一般使用者，但除錯時很有用。

## 6. 什麼時候抓即時報價

正確做法：

- 使用者進入某檔股票頁面或圖表時抓一次。
- 使用者切換股票時抓一次。
- 使用者按「更新」按鈕時抓一次。
- 首頁如果停在某一檔重點股票，可以只抓那一檔。

不建議做法：

- 不要一進首頁就抓全部股票的即時報價。
- 不要每秒輪詢所有股票。
- 不要把即時報價資料寫入 `stock_daily_bars`，除非後端已明確設計盤中資料表。

原因：

- 官方資料來源不是給專題系統大量掃描用的。
- 大量請求會拖慢前端，也可能讓官方來源拒絕服務。
- 本專題需要的是「合理展示」與「資料庫流程」，不是高頻行情系統。

## 7. 後端 API 清單

backend base URL：

```txt
http://localhost:3001
```

| Route | 用途 | 前端使用情境 |
|-------|------|--------------|
| `GET /api/health` | 檢查 backend 與資料庫狀態 | 啟動檢查 |
| `GET /api/market/db-bars/:code` | 取得某檔歷史 K 線 | K 線圖與 MA 指標 |
| `GET /api/market/db-bars/:code?refresh=1` | 強制重新抓歷史 K 線 | 測試或資料異常時 |
| `GET /api/quote/:code` | 取得某檔準即時報價 | K 線圖上方報價條 |
| `GET /api/market/stocks` | 取得 TWSE OpenAPI 股票清單 | 股票列表或驗證 |
| `GET /api/market/stocks/:code` | 取得單檔最新交易日資料 | 股票基本資訊 |
| `GET /api/market/daily-bars` | 取得最新交易日 K 資料 | 最新日資料，不是長期 K 線 |
| `GET /api/market/import-preview` | 預覽匯入資料庫格式 | 後端與資料庫檢查 |
| `GET /api/twse/*` | 透過 backend proxy 查 TWSE 原始資料 | 除錯與補資料 |

## 8. 前端目前已完成的整合

目前前端已經做到：

- `StockChart.vue` 會用 `getDbBars(stock.code)` 讀取後端歷史 K 線。
- `StockChart.vue` 會在股票切換時重新讀取該股票歷史資料。
- `StockChart.vue` 會在資料可用時顯示「來源:資料庫(TWSE 歷史)」。
- `LiveQuote.vue` 會在進入股票圖表時呼叫 `getQuote(code)`。
- `LiveQuote.vue` 會在股票切換時重新抓報價。
- `LiveQuote.vue` 有手動更新按鈕。
- `LiveQuote.vue` 沒有每秒無限輪詢，這是正確的資源控制。

## 9. 前端組員容易搞錯的地方

### 錯誤 1：把 `getMappedDailyBarByCode` 當成歷史 K 線來源

這不適合。它主要來自 TWSE OpenAPI 最新交易日資料，不是完整歷史。

正確做法：

```js
import { getDbBars } from '@/services/twseApi'
```

### 錯誤 2：把 `getQuote` 回傳資料塞進 K 線陣列

這不建議。即時報價不是正式日 K。

正確做法：

```txt
K 線圖繼續用 getDbBars
報價條另外用 getQuote
```

### 錯誤 3：首頁一次抓全部股票報價

這不務實，也不必要。

正確做法：

```txt
使用者正在看哪一檔，就抓哪一檔。
```

## 10. 對外說明建議

如果老師或組員問「你們的股票資料是不是真實的」，建議回答：

```txt
K 線圖使用後端從 TWSE 官方 STOCK_DAY 抓取的歷史日線，並存入 SQL Server。
MA 指標由前端根據這些歷史 OHLCV 計算。
圖表上方的報價條使用 TWSE MIS 的單檔準即時報價，而且只有使用者查看該股票時才抓，後端有 20 秒快取。
```

不要說：

```txt
我們有完整逐筆即時行情。
```

原因：

- 目前是準即時報價，不是付費等級逐筆成交資料。
- 專題展示這樣已經足夠，硬說成完整即時行情反而容易被追問到無法自圓其說。
